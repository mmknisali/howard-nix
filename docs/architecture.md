# Architecture

Technical details about how this NixOS configuration works.

## Flakes Overview

This is a **flake-based** NixOS configuration. Flakes provide reproducible, declarative system configurations.

### Key Files

| File | Purpose |
|------|---------|
| `flake.nix` | Entry point, defines inputs and outputs |
| `flake.lock` | Locked versions of all dependencies |
| `hosts/howard/default.nix` | Main system configuration |

## Nixpkgs Inputs

Two nixpkgs inputs are available:

```nix
nixpkgs.url = "github:NixOS/nixpkgs/nixos-24.11";  # Stable
nixpkgs-unstable.url = "github:NixOS/nixpkgs/nixos-unstable";  # Unstable
```

- Use `pkgs` (stable) for most packages
- Use `pkgs-unstable` for packages not in stable or newer versions

## Module System

### How It Works

NixOS uses a module system where:
1. Each Nix file defines `options` (configuration points)
2. Each file defines `config` (what gets set when options are enabled)

### Module Hierarchy

```
flake.nix
  └── nixosConfigurations.howard
        └── modules = [ ./hosts/howard ]
              ├── default.nix (imports others)
              ├── hardware-configuration.nix
              ├── config/starship/starship.nix
              └── config/zsh/zsh.nix
```

### Importing Modules

In `default.nix`:
```nix
imports = [
  ./hardware-configuration.nix
  ./config/starship/starship.nix
  ./config/zsh/zsh.nix
];
```

## Directory Structure

```
hosts/howard/
├── default.nix              # Main config, imports modules
├── hardware-configuration.nix  # Generated by nixos-generate-config
├── keys                    # SSH public keys
└── config/
    ├── starship/
    │   ├── starship.nix    # Nix module to enable starship
    │   └── starship.toml    # Starship settings
    └── zsh/
        ├── zsh.nix         # Nix module to enable zsh config
        ├── .zshenv         # Zsh environment (sourced always)
        ├── .zshrc          # Zsh interactive config
        ├── user.zsh        # User customizations
        ├── prompt.zsh      # Prompt config
        ├── plugin.zsh     # Plugin config
        ├── conf.d/         # HyDE configurations
        ├── functions/      # Custom functions
        └── completions/    # Custom completions
```

## Zsh Integration

The zsh configuration is handled by:

1. **`zsh.nix`** (Nix module):
   - Copies `config/zsh/` to `/etc/zsh`
   - Sets `ZDOTDIR=/etc/zsh`
   - Enables zsh

2. **HyDE** (zsh framework):
   - Loaded via `conf.d/00-hyde.zsh`
   - Manages prompt, plugins, completions
   - User customizations go in `user.zsh`, `prompt.zsh`, `plugin.zsh`

## Configuration Patterns

### Adding a Package

```nix
environment.systemPackages = with pkgs; [
  package-name
];
```

### Enabling a Service

```nix
services.servicename = {
  enable = true;
  # settings...
};
```

### Using Lib Helpers

```nix
# Set default value (can be overridden)
lib.mkDefault true

# Force a value (cannot be overridden)
lib.mkForce "value"

# Conditional configuration
lib.mkIf condition config
```

## Best Practices

1. **Never edit `hardware-configuration.nix`** - it's auto-generated
2. **Commit `flake.lock`** - ensures reproducible builds
3. **Test with dry-build** before applying
4. **Use relative paths** when possible (works with flakes)
5. **Keep secrets out** - use `keyFiles` or external secrets management
